## What Worked Well

### Workflow loop (axis 1)

- **In-band fix loop kept work in flow.** Every Codex finding shipped as a follow-up commit in the same session as the dev work. Zero deferral to backlog. 1-2 commits per story average (8 commits / 4 code-shipping stories = 2.0).
- **TDD discipline held across techniques.** Stories 2.2/2.3/2.4 logged explicit RED states in dev notes. Story 2.5 (bash) used schema-diff against golden file as the contract test — different mechanism, same gate.
- **Story-create cross-references previous review findings.** Each subsequent story's "Previous Story Intelligence" section explicitly forward-warned against patterns from the prior fix (e.g., 2.4 warned about staleness defects after 2.3 R1, 2.5 warned about analogous gaps in bash after 2.4 R1).

### Same-session vs cross-session decisions (axis 2)

- **Same-session create+dev was the right call.** Cache warm, project-context already loaded, budget-efficient. 5h budget progression: 42% (2.2 dev start) → 54% (2.3) → 62% (2.4) → 71% (2.5) — a single session held all 4 code-shipping stories.
- **Cross-session Codex review caught what same-session never could.** Story 2.4 R1 is the proof: implementation matched the architecture pin perfectly, but the pin itself was wrong. Same-session reviewer would have rationalized the spec. Different LLM with no implementation context flagged the spec-vs-reality drift.
- **Two orthogonal axes proved necessary.** Same-session catches spec↔implementation drift. Cross-session catches spec↔reality drift. Both required.

### Codex finding quality (axis 4)

- **5/5 valid, zero bikeshed.** Every finding was a contract or specification gap, none were stylistic preferences.
- **Findings spanned all three contract layers:** configuration (2.2 R1 — docker build chain), runtime lifecycle (2.3 R1 — scope race), API/spec drift (2.4 R1 — Sentry frame matching), API surface (2.5 R1 — paginated shape), and error semantics (2.5 R2 — exit code map).
- **One P1 caught before production.** Story 2.5 R1 (paginated `{items}` shape crash) would have shipped as silent breakage; the homelab GlitchTip masks the bug by returning plain arrays for sub-page responses.

## What Didn't Work / Friction Points

### Story-create discovery gap (axis 1)

- **Sibling configs missed.** Story 2.2 plan listed `vite.config.ts` but not `vitest.config.ts`. Both have parallel `define` blocks. The dev hit `ReferenceError: __BUILD_HOST__ is not defined` at GREEN, then had to mirror the change manually. Caught in the same session but added round-trip cost.

### Pre-existing issues escalating without promotion (axis 3 — deeper analysis)

| Issue | Flagged in | Pain trajectory in Epic 2 |
|---|---|---|
| `infra/.env` line 17 multi-line value chokes `source` under `set -e` | 2.1, 2.2, 2.3, 2.4, 2.5 | **Escalating**: discovery → annoyance → tooling friction → workaround. Story 2.5 added defensive `set +e ... source ... 2>/dev/null ... set -e` guard because deploy-critical script could not crash on env load. Workaround paid, real fix still pending. |
| `apps/web/src/ui/custom/CardCarousel.test.tsx` — 3 React rendering test failures | 2.2, 2.3, 2.4, 2.5 | **Stable noise**: every `npm run test` raports "X passed / 3 failed (pre-existing)". Sygnał degradation — accepting failing tests dulls regression sensitivity. |

- **Diagnosis:** BMAD's "no silent scope creep" rule is intentional and correct, but it lacks an **escape valve**. After N flags, the system should promote the issue to a triage candidate, not just re-flag it. The threshold for "N" was undefined — five flags had to accumulate before the pattern surfaced in retro.

### Documentation drift (axis 4)

- **Story 2.4 R1 fix shipped as code, not as architecture doc rewrite.** Decision H step 1 in `_bmad-output/planning-artifacts/architecture.md` still says "match `event.request?.url`" but the implementation now matches both request URL AND stack frame filenames. The next story reading architecture.md would inherit the wrong pin.

## Decisions Made in Retrospective

### Triage of pre-existing issues (Project Lead's call)

All three issues promoted to quick-dev queue, sequential ordering, atomic commits, **no cleanup-PR consolidation**:

| # | Issue | Quick-dev priority | Owner | Effort |
|---|---|---|---|---|
| QD-1 | `infra/.env` line 17 multi-line value real fix (replaces 2.5 workaround) | NEXT | Ezop (format decision), Amelia (impl) | 10-30 min |

| QD-3 | `CardCarousel.test.tsx` — diagnose (test bug vs component bug) + fix | AFTER QD-2 | Amelia (diagnosis may need Project Lead decision on fix scope) | 30-90 min |

Rationale for ordering: QD-1 first because workaround is already paying technical-debt interest in deploy-critical script. QD-2 second because zero behavior change (atomic guardrail). QD-3 last because it requires diagnosis and benefits from a clean post-QD-1+QD-2 baseline so the test failure surface is unambiguous.

### Threshold for promoting pre-existing issues (Project Lead approved with conditions)

- **Rule:** A pre-existing issue flagged in **3 separate stories' Dev Agent Records** auto-creates a candidate entry in `_bmad-output/triage-backlog.md` (gitignored).
- **Important nuance per Project Lead's correction:** the threshold creates a **candidate**, NOT a quick-dev story. Promotion to quick-dev remains Project Lead's manual call. This preserves Project Lead control over backlog content.
- **Calibration window:** the threshold itself is tentative. After 2 more retros (Epic 4 + Epic 5 close-out), the team re-evaluates whether "3" is the right number — could be too low (noise) or too high (more workarounds accumulate before promotion).

## Key Takeaways

1. **Cross-session Codex review is non-substitutable.** Same-session review cannot catch spec-vs-reality drift. Story 2.4 R1 is the canonical proof.
2. **The `create+dev (same-session) → Codex (cross-session) → fix (same-session)` loop converges on 1-2 commits per story.** Healthy throughput, no backlog accumulation.
3. **Pre-existing issues without a promotion mechanism escalate into workarounds.** Story 2.5 had to write a defensive guard around a broken env file. The triage-backlog threshold closes this loop.
4. **Codex catches contract gaps, not style.** 5/5 findings were architectural or specification issues. The review can be relied upon.
5. **Doc drift is real.** Code fix without corresponding architecture.md update is a time-bomb for the next story reading the pin.
