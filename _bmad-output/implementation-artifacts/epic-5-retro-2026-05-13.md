## What went well

1. **Adversarial review in product-brief workflow surfaced 2 P0 + 5 P1 BEFORE any code was written.** The v1 brief's Stylelint `color-function-notation: 'legacy'` pin would have file-wide-flipped 17 declarations against the operator's intent. The token-file split (Decision B) emerged from the P0-1 finding. Without the review the implementation would have either auto-fixed wrong (mass change no-one asked for) or generated 17 silence-disable comments (theatre — the exact anti-pattern Initiative 3 was created to eliminate).

2. **Phase A audit revealed scope creep BEFORE Phase B started.** Brief assumed 5 violations (3 dialog/sheet + 2 viewer overlays). Audit found 11: CardCarousel × 3 and ModelGallery × 3 mirroring each other. Story 5.10 absorbed the gallery work cleanly via shared `--color-gallery-control` tokens (one token-add, two consumer sites — pattern brief recommended). Without Phase A this would have been a Phase B surprise + scope-extension.

3. **Layered lint paid off immediately.** In-repo `no-restricted-syntax` rule (Decision C) caught dialog.tsx + sheet.tsx + ModelGallery + CardCarousel violations cleanly during Phase B. `@poupe` plugin enhancement layer (gated on Phase A audit being clean) was deferred — the in-repo rule alone delivered the full warning surface. Plugin remains optional (pre-1.0 single-maintainer; if it goes silent, no impact on critical path).

4. **The hook design was tested by its own first invocation.** Story E5.11 (baseline regen) was the first commit to touch `apps/web/tests/visual/__snapshots__/`. The hook initially REJECTED the commit because pre-commit fires before `.git/COMMIT_EDITMSG` is updated by `git commit -F` — caught a real design flaw in the hook script. Fixed inline (pre-commit emits warning only; commit-msg does the strict enforcement). The dual-hook pattern is now actually working as intended, with the design fault discovered + corrected in-band.

5. **Subagent delegation for E5.12 worked.** Four sub-stories (5.12a–d) implemented sequentially by a single subagent in ~6.5 min total — would have taken 1–2 hours in main context. Subagent found navigation paths the main context didn't have time to investigate (e.g., StatusPopover trigger via `<span>` wrapper because DropdownMenuTrigger uses `nativeButton=false`; ModelHero admin entry via "Akcje modelu" kebab; Edytuj tagi via separate pencil button).

6. **Phase C-early-before-Phase-B sequencing held.** Lint + Stylelint + axe (E5.4–E5.6) landed before remediation source changes (E5.7–E5.10). During Phase B remediation, every commit was lint-validated against the new rules. Zero churn — fixes landed once, verified once.

## What surprised us

1. **The "14 skips" was 4 source skip-statements × matrix expansion.** Phase A audit (E5.2) found that the 14 baseline-skip count from the brief was matrix-expanded test runs, not 14 distinct skip statements. 4 source `test.skip(...)` / `describe.skip(...)` blocks × 4 projects = 14 runs. 12 of the 14 are placeholder skips for not-yet-implemented features (admin-thumbnail-flow + files-tab-admin + catalog-detail-admin describe.skip placeholders waiting for Slices 3D/3E). The "silent quality hole" the brief worried about doesn't exist — the skips are intentional. Orphan PNGs on those describe.skip specs (12 PNGs total) should be deleted in a follow-up cleanup.

2. **Dark-mode visual tests did NOT fail after Phase B tokenization.** The 14 visual-regression failures from Phase B token replacement landed entirely in light-mode variants. Dark variants pass unchanged because the new token values are theme-invariant near-black, which matches the dark-theme rendering of the old hardcoded rgba. Light-theme rendering is where the contrast diff was always visible — confirming the brief's diagnosis that this is a LIGHT-mode-specific defect class.

3. **Visual-coverage hook needed pre-commit lenience.** Initial pre-commit hook design assumed `.git/COMMIT_EDITMSG` was current at pre-commit time. It is NOT — it contains the PREVIOUS commit's message until the new message is finalized. The corrected pattern: pre-commit emits a warning naming the staged PNGs; commit-msg (which runs after the message is set) does strict enforcement. This is now a 2-hook chain, not a single hook.

4. **Husky monorepo gotcha:** `npx husky init` from a subdirectory can't find `.git`. Fix: set `core.hooksPath` explicitly via `npm run prepare`. AND the prepare script must tolerate non-git environments (Docker builds without git binary, CI runners) — discovered via deploy failure on first try. Fix-up: `git rev-parse --is-inside-work-tree || true` guard.

5. **`.codex/` gitignore meant Decision J's "committed" artifact wasn't actually committed.** Architecture extension assumed `.codex/` was already a committed surface (because other agent prompts live there in some repos). Reality: it was gitignored as "operator-local orchestration logs". Fix-up: carved out `.codex/review-prompts/` and `.codex/bin/` via gitignore negation. Architecture decision held; gitignore caught up.

## What we'd do differently

1. **Verify .gitignore status of any architectural decision-named path BEFORE committing the architecture.** Would have caught the `.codex/` gotcha in Stage 4 adversarial review instead of at implementation time.

2. **Pre-design hook scripts against the actual git lifecycle event timing.** The pre-commit vs commit-msg distinction (`.git/COMMIT_EDITMSG` availability) should have been in the architecture extension's Decision F or G description, not discovered live.

3. **Audit reports (Phase A) should declare which assumptions in the brief were CONFIRMED vs REFUTED-with-diff up front, not buried in conclusions.** Brief assumed UserMenu was uncovered; matrix found it covered via agents-info-dialog.spec.ts. Brief assumed 10 surface gaps; matrix found 22 (Radix instances counted, not unique component types). Restructuring those callouts to the top of the audit makes Phase B story scoping faster.

4. **Skip the "Phase A operator visual review" assumption for autonomous-mode runs.** Story E5.2 spec required operator eye-review of all 82 PNGs. In autonomous mode that's not feasible. The pragmatic compromise (most baselines marked `unverified-defer-to-operator`) is fine, but it should have been the documented default in the spec, not an inferred adaptation.

5. **E5.12 sub-split should have been 3 sub-stories not 4.** 5.12a (FilterRibbon Selects) is small enough that it could have folded into 5.12b (form/dialog open states). The sub-split was driven by Phase A matrix groupings but the actual baseline counts (6 vs 16 vs 16 vs 16) show 5.12a is overhead-heavy.

## Procedural lessons (for future BMAD initiatives)

1. **Adversarial review before PRD route is high-leverage.** Cost: ~3 min of subagent time. Surfaced 2 P0 + 5 P1 + many P2/P3 fixes BEFORE code, BEFORE PRD, BEFORE architecture. Document this as a standard step in the bmad-product-brief workflow.

2. **Phase sequencing matters more than story count.** Epic 5 had 17 stories but only 4 phase boundaries (A → C-early → B → C-prev). The phase ordering was the load-bearing architectural decision; getting it right at planning time prevented re-work.

3. **Hooks must be tested against the actual git lifecycle, not the documented one.** The pre-commit-vs-commit-msg distinction is subtle and not obvious from reading husky docs. Smoke-test the hook against `git commit -F`, `git commit -m`, `git commit` (interactive editor), and `git commit --amend` before claiming hook stories done.

4. **Architecture decisions naming external paths (`.codex/`, `.husky/`, etc.) must verify gitignore status.** A "committed artifact" decision is invalid if the path is gitignored.

5. **Subagent delegation for multi-file boilerplate tasks (writing N similar specs) is ~10× faster than main context.** Use subagents for E5.12-class work; use main context for single-file critical-path decisions.
